Balance Sweep

Build a npm run fuzz balance-sweep / search command → copy → run in a terminal. Dev-only GUI over the fuzz CLI (tests/fuzz/cli.ts); the search runs in Node, so this emits a command to paste, not a URL. ← Tools

Mode

Sweep grid

min : max : steps — steps is the point count (inclusive). 0.625 : 0.625 : 1 pins one point.

Each grid point runs a weight search (best-achievable) + the pure-random / greedy baselines (the skill gradient) + per-archetype telemetry. Writes tests/fuzz/output/balance-sweep.csv + .report.txt.

Search

Random-searches the scored-strategy weight space for the best- achievable win rate at the current config; writes tests/fuzz/output/best-strategy.json (re-runnable via --strategy=…json). Tune config first via balance-sweep.

Run options

--hops overrides the tier/preset run length (cheap full-length reads). --roster forces the starting team (evaluate an archetype the search won't recruit). --jobs > 1 fans the search across child processes (vector-level sharding, byte-identical results) — works in both modes. Same sampler seed + grid + tier reproduces a run.

Command

Paste into a terminal at the project root (needs npm install). The fuzz CLI runs standalone — no dev server needed.
