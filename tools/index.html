ASCIIbattler Dev Tools

dev-only · served by npm run dev · none ship in dist/ ← Game

Config editors — author & save to config/*.json

Sector Editor

Run containers — each sector's layout pool (weighted, hop-gated), theme, and length. Live weighted-pool preview.

config/sectors.json

Layout Editor

Paint battlefields — walls, water, half-cover, fire/healing tiles, spawn regions, theme — with a live grid canvas.

config/layouts.json

Archetype Editor

Tune unit stats, abilities, targeting, and growth rates with a live derived-stat / to-hit / cadence preview.

config/units.json

Encounter Editor

Author authored fights — name, health pool, kind, layout fit-filter, and the wave-list grammar (wave / pick / loop / stages) — with a live turn-by-turn resolution preview that shares the real resolvers.

config/encounters.json

Reward Table Editor

Author weighted reward tables — bits ranges, daemon grants, dormant packet entries — with a live draw-chance preview and encounter cross-references.

config/rewards.json

Attack Editor

Author ability definitions — name, cooldown, range, target selector, timeline, and the effect-op tree (damage / heal / status / chain / summon) — with live schema validation and a structure preview.

config/abilities.json

Run & balance harnesses

Run Launcher

Launch the game with a specific run config — short runs, a forced layout, a leveled or hand-picked starting roster, a chosen daemon.

drives RunConfig · opens the game with a query string

Balance Sweep

The GUI front-end for the headless fuzz harness — run difficulty sweeps and read the per-archetype / pool / hop telemetry.

front-end for npm run fuzz

Prototypes

Map-Gen Prototype

A standalone sandbox for the procedural node-map generator — visualize the layered DAG shapes a seed produces.

node-map generate() sandbox
Each tool is a standalone Vite page with no rollupOptions.input entry, so it is served only by the dev server and never bundled into a production build. The config editors save straight to disk through the dev-only /__save-config endpoint (vite.config.ts); an open game tab hot-reloads the new values.
